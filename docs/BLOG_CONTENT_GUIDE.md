# Руководство по наполнению блога

Контент статей блога хранится в формате JSON массива блоков. Это позволяет гибко управлять версткой и добавлять интерактивные элементы.

## Структура контента

Поле `content` в переводе статьи (`blog_article_translations`) должно содержать массив объектов. Каждый объект представляет собой отдельный блок контента.

### 1. Заголовки (Heading)

Используются для структурирования текста. Автоматически генерируют оглавление статьи.

```json
{
  "type": "heading",
  "level": 2,
  "text": "Заголовок секции"
}
```

- **level**: Уровень заголовка от 1 до 6 (обычно используются 2 и 3, так как H1 - это название статьи).
- **text**: Текст заголовка.

### 2. Параграфы (Paragraph)

Основной текстовый блок. Поддерживает форматирование через HTML-теги.

**Поддерживаемые стили:**

- **Жирный**: `<b>текст</b>` или `<strong>текст</strong>`
- _Курсив_: `<i>текст</i>` или `<em>текст</em>`
- ~~Зачеркнутый~~: `<s>текст</s>` или `<del>текст</del>`
- <u>Подчеркнутый</u>: `<u>текст</u>`
- [Ссылка](https://example.com): `<a href="https://example.com">текст</a>`
- `Код`: `<code>текст</code>`
- Перенос строки: `<br>`

```json
{
  "type": "paragraph",
  "text": "Вы можете выделить <b>важные моменты</b> или использовать <i>курсив</i> для терминов. <br> А также добавлять <a href='https://google.com'>ссылки</a>."
}
```

- **text**: Текст параграфа с HTML-тегами.

### 3. Списки (List)

Маркированные или нумерованные списки.

```json
{
  "type": "list",
  "ordered": false,
  "items": ["Первый пункт", "Второй пункт", "Третий пункт"]
}
```

- **ordered**: `true` для нумерованного списка (1, 2, 3), `false` для маркированного (точки).
- **items**: Массив строк (пунктов списка).

### 4. Цитаты (Quote)

Выделенные блоки с цитатами или важными мыслями.

```json
{
  "type": "quote",
  "text": "Образование — это самое мощное оружие, которое вы можете использовать, чтобы изменить мир.",
  "author": "Нельсон Мандела"
}
```

- **text**: Текст цитаты.
- **author**: (Опционально) Автор цитаты.

### 5. Изображения (Image)

Блок изображения с возможностью настройки ширины.

```json
{
  "type": "image",
  "url": "/uploads/university-campus.jpg",
  "alt": "Кампус университета",
  "caption": "Вид на главный корпус",
  "width": "wide"
}
```

- **url**: Ссылка на изображение (можно использовать внешние ссылки или локальные пути).
- **alt**: Описание изображения для доступности и SEO.
- **caption**: (Опционально) Подпись под изображением.
- **width**: (Опционально, по умолчанию 'standard')
  - `standard`: Стандартная ширина, с отступами (как текст).
  - `wide`: Увеличенная ширина, выступает за границы текста (если позволяет дизайн).
  - `full`: На всю ширину карточки/экрана (игнорирует отступы контейнера).

### 6. Разделитель (Divider)

Горизонтальная линия для визуального разделения секций.

```json
{
  "type": "divider"
}
```

### 7. Отступ (Spacer)

Пустой блок для добавления вертикального пространства между элементами. Используйте для визуальной разрядки контента.

```json
{
  "type": "spacer",
  "size": "md"
}
```

- **size**: Размер отступа.
  - `sm`: Маленький (1rem / 16px)
  - `md`: Средний (2rem / 32px) - _по умолчанию_
  - `lg`: Большой (3rem / 48px)
  - `xl`: Очень большой (4rem / 64px)

## Пример полной структуры

```json
[
  {
    "type": "paragraph",
    "text": "Введение в статью..."
  },
  {
    "type": "heading",
    "level": 2,
    "text": "Первый раздел"
  },
  {
    "type": "image",
    "url": "https://example.com/image.jpg",
    "alt": "Описание",
    "width": "full"
  },
  {
    "type": "spacer",
    "size": "sm"
  },
  {
    "type": "paragraph",
    "text": "Текст под изображением."
  },
  {
    "type": "divider"
  },
  {
    "type": "quote",
    "text": "Заключительная мысль."
  }
]
```
